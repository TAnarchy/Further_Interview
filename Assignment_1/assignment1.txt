1.3 (Logging Questions). I chose 2 ways to log the errors. Via console.log and via DataDog. DataDog is the primary logging tool here. DataDog or a similar logging tool would be my choice for any sort of logging. If I was using a server-side language, 
I might have wrote to a local log file as well. Having DataDog makes troubleshooting much easier, I can easily identify issues that happened at specific days or times. 
It is especially useful if I am having difficulty reproducing the issue that a client is experiencing, or if the issue is intermittent. 
Console.log allows me to see issues in real-time, and is there as a back-up if DataDog fails for some reason

Directions to run:

1. In order for DataDog logging to work properly, the application must run on a local server. 
2. The Assignment1_HTML.html file and the assignment1.js must be in the same folder for the application run.
3. I tested this with the Python local server, you can run it by going in the working directory via Linux console and running the command:
python -m http.server 8000.
4. The application will then be live on http://localhost:8000/Assignment1_HTML.html
5. The application will work outside of a local server, the DataDog logging will just not work. Of course, this won't reflect in the application in any way, other then console errors about the failure.
